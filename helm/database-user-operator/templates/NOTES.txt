IMPORTANT: The operator requires AWS permissions to access Secrets Manager.
AWS credentials must be configured externally via:
  - IRSA (IAM Roles for Service Accounts) - recommended for EKS
  - EC2 Instance Profile
  - Environment variables or other AWS SDK credential chain methods

Created database credentials are always stored in AWS Secrets Manager.

To create a Database resource, see examples at:
  https://github.com/opzkit/database-user-operator/tree/main/config/samples

Example:
  kubectl apply -f - <<EOF
  apiVersion: database.opzkit.io/v1alpha1
  kind: Database
  metadata:
    name: example-database
  spec:
    engine: postgres
    databaseName: myapp_db
    connectionStringSecretRef:
      name: postgres-admin-connection
      key: connectionString
  EOF
